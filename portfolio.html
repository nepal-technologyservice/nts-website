MIGREASE

The Project

Migrease is a web platform built entirely in-house by NTS. It was the first major production deployment NTS undertook — and it was used to prove the approach: build the right way from day one, regardless of the pressure to move fast and cut corners.

Architecture

The infrastructure is managed entirely in Terraform — no manual console configuration, no infrastructure that lives only in someone's head. Every resource (Lambda functions, S3 buckets, CloudFront distributions, API Gateway, RDS) is versioned, documented, and reproducible from code.

The backend runs on AWS Lambda with a Node.js/TypeScript runtime — serverless where it makes sense, with proper cold-start mitigation and function isolation. The frontend is a Next.js application served via CloudFront with edge caching, delivering sub-second load times.

CI/CD & Deployment

Every code change goes through a GitHub Actions pipeline: automated tests run first, build artifacts are created and validated, and deployments happen only on passing pipelines. Staging and production environments are separate, with environment-parity maintained through Terraform workspaces.

Deployment to Lambda is automated — no FTP, no manual zipping, no SSH into a server. The entire process from merged PR to production takes under 3 minutes.

Security

IAM roles are scoped to least-privilege. No broad permissions, no shared credentials. Secrets are managed through AWS Secrets Manager — not environment variable files committed to the repo. The API includes rate limiting, input validation, and proper error responses that don't leak internal information.

Documentation

Full API documentation was written in OpenAPI format. Infrastructure decisions are documented in ADRs (Architecture Decision Records). The README covers local setup, environment configuration, deployment process, and common debugging scenarios. Any developer should be able to pick this up without asking us how it works.
